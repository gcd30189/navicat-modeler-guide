## 核心功能拆解

先说几个实际工作中最常用的能力，不讲概念，只说能干什么。

**三层模型体系，随意转换。** 从概念模型（业务实体关系层）→ 逻辑模型（字段和主外键定义）→ 物理模型（对应真实数据库表结构），三层可以互相转换，不用每层重新画。项目初期画概念图，细化后转逻辑层，上线前转物理层，全程不断线。

**逆向工程。** 已经有现成数据库？直接连上去，一键生成 ER 图。老系统接手的时候特别有用，接手的库文档缺失，逆向工程拉出来一看，表结构和关系一目了然。

**数据库比对与同步。** 模型改了，和生产库之间有差异？比对功能帮你看清差异在哪，自动生成同步脚本，不用手动比。DBA 做变更发布的时候，这个功能能救命。

**SQL 脚本导出。** 模型设计好，一键导出完整 SQL 脚本，包含完整性规则、注释、字符集等细节，省的是重复手写的功夫。

**Navicat Cloud 协作。** 模型文件同步到云端，团队成员实时访问，不用再靠传文件维护版本一致。
